控制论（Cybernetics）由诺伯特·维纳在 1948 年提出，研究**复杂系统中的目标导向行为和反馈调节**。它同样适用于理解 AI 对话的动态。

**核心概念：负反馈与正反馈**

负反馈（Negative Feedback）：减少偏差，趋向目标。

```
期望状态 → 控制器 → 执行器 → 被控系统
              ↑                    ↓
              ←←←← 传感器 ←←←←←←←←
```

恒温器是典型例子：温度偏高 → 关闭加热 → 温度下降 → 恢复稳态。

正反馈（Positive Feedback）：放大偏差，趋向极端。

**在 Claude 对话中的体现**

对话本身就是一个控制系统：
- **目标**：完成任务
- **控制器**：Claude 的推理
- **执行器**：输出的文字
- **传感器**：你的反馈（追问、纠正、确认）

每次你的回复都是一次反馈信号。关键在于信号质量：

| 反馈质量 | 示例 | 效果 |
|---------|------|------|
| 负反馈（纠偏）| "这里不对，实际上..." | 快速收敛到正确答案 |
| 正反馈（放大）| "对对对！继续" | 深化当前方向 |
| 零反馈 | "嗯" | 系统盲目漂移 |

**稳态（Homeostasis）与模型漂移**

长对话中，Claude 会发展出对你偏好的"稳态模型"——一种对"你想要什么"的内部表示。这有好处也有风险：

好处：不需要每次都重新解释背景。

风险：如果稳态建立在错误的基础上，Claude 会"礼貌地同意"而非纠错。信号：当 Claude 的回答开始越来越像你说的话时，要主动引入对抗性反馈。

**等终性（Equifinality）**

控制论中的等终性原理：不同的初始状态，可以通过不同路径，到达相同的最终状态。

对提示词工程的启发：**同一个目标可以用完全不同的提示词到达**。如果一种提法行不通，不要只是重复同样的话，换一个起点：

```bash
# 路径 1 失败：直接要答案
claude "这段代码的 bug 在哪里？"

# 路径 2：让 Claude 描述代码行为
claude "这段代码在输入 X 时会做什么？"
# → Claude 描述过程中自己发现 bug
```

**控制延迟与预测**

真实控制系统有延迟问题——反馈信号到达时，系统已经变了。AI 对话的"延迟"是理解延迟：Claude 对你的意图可能有一拍的滞后理解。

对策：减少延迟的最有效方法是**提前说明终态**（期望的最终结果），而不是一步步给指令等反馈。这等价于前馈控制（Feedforward Control）——在误差出现前就进行修正。

```bash
# 反馈控制（有延迟）
claude "帮我写函数"
→ 输出不对
→ "改一下"
→ ...

# 前馈控制（无延迟）
claude "帮我写一个函数，最终形态是：
- 签名：func Parse(s string) (Result, error)
- Result 包含 Name string 和 Score float64
- 无效输入返回具体错误类型（不是 error string）
- 单元测试要覆盖空字符串和负数"
```

控制论的核心洞见：**你和 AI 共同构成一个反馈系统，系统的表现取决于你发出的信号质量**。单向灌输问题，等待完美答案——这不是控制，这是开环。
