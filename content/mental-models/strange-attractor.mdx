      <div class="formula">混沌系统轨迹：不可预测单次路径，但永远在有界区域内
LLM 输出：不可完全控制单次输出，但可以定义"不可能区域"

好的 system prompt ≠ 控制每次输出
好的 system prompt = 定义输出的可能空间（吸引域）

原则：约束"什么绝对不出现" > 规定"必须出现什么"</div>

<pre><span class="cm">❌ 错误：试图控制每次输出（脆弱，过拟合）</span>

你是一个社媒文案 agent。
每次输出必须：
- 第一句以数字或问题开头
- 第二句解释背景
- 第三句给出行动号召
- 结尾用 3 个 emoji
- 字数在 180-220 字之间
...（30条规则）

<span class="cm">→ 对某类 topic 完全失控，规则相互冲突</span>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

<span class="cm">✅ 正确：定义吸引域边界（鲁棒，泛化性强）</span>

## 引力中心（目标区域）
你生成的内容在语气上贴近真实用户，不像 AI，
在信息上聚焦有价值的洞察，不是空泛的感叹。

## 排斥边界（绝对不出现）
- 不使用"赋能/赋权/生态"等营销黑话
- 不使用 5 个以上 emoji（视觉噪音）
- 不承诺具体数字收益（合规风险）
- 不提及竞争产品名称
- 不超过 280 字

## 自由区间（故意不规定）
- 开头方式：自由，根据内容选择最自然的
- 句子数量：自由，够清晰就行
- 话题角度：自由，选最有趣的切入点</pre>

<pre><span class="cm">// 吸引域验证器：检查输出是否在边界内</span>
<span class="kw">interface</span> <span class="var">AttractorBoundary</span> {
  hardRules: {               <span class="cm">// 排斥边界：违反 = 重新生成</span>
    maxLength: <span class="kw">number</span>;
    forbiddenPatterns: RegExp[];
    requiredElements?: <span class="kw">string</span>[];
  };
  softGuidelines: <span class="kw">string</span>[];  <span class="cm">// 引力中心：写在 prompt 里，不做硬检查</span>
}

<span class="kw">const</span> xerpaAgentBoundary: <span class="var">AttractorBoundary</span> = {
  hardRules: {
    maxLength: <span class="num">280</span>,
    forbiddenPatterns: [
      <span class="str">/赋能|赋权|生态/g</span>,
      <span class="str">/保证.*%收益|承诺.*倍/g</span>,
      <span class="str">/(competitor1|competitor2)/gi</span>,
      <span class="str">/[^\S\r\n]{2,}/g</span>,  <span class="cm">// 多余空格</span>
    ],
  },
  softGuidelines: [
    <span class="str">'语气真实，不像营销文案'</span>,
    <span class="str">'优先用具体数据替代形容词'</span>,
  ],
};

<span class="kw">async function</span> <span class="fn">generateWithBoundary</span>(
  prompt: <span class="kw">string</span>,
  boundary: <span class="var">AttractorBoundary</span>,
  maxRetries = <span class="num">3</span>
): <span class="kw">Promise</span>&lt;<span class="kw">string</span>&gt; {
  <span class="kw">for</span> (<span class="kw">let</span> i = <span class="num">0</span>; i &lt; maxRetries; i++) {
    <span class="kw">const</span> output = <span class="kw">await</span> <span class="fn">callClaude</span>(prompt);
    <span class="kw">const</span> violation = <span class="fn">checkBoundary</span>(output, boundary.hardRules);
    
    <span class="kw">if</span> (!violation) <span class="kw">return</span> output; <span class="cm">// 在吸引域内</span>
    
    <span class="cm">// 把违规信息加回 prompt，让模型自我修正</span>
    prompt += <span class="str">`\n\n[修正提示] 上次违规: ${violation}，请重新生成`</span>;
  }
  <span class="kw">throw new</span> Error(<span class="str">'无法在边界内生成合格内容'</span>);
}</pre>

<pre><span class="cm">## System Prompt 结构（吸引域三层模板）

### 第一层：引力中心（写在 prompt 里，不做硬检查）
- 目标语气/风格方向
- 好的输出"看起来像什么"

### 第二层：排斥边界（硬约束，程序化检查 or 明确列出）
- 禁止词汇：[根据业务填写，如"赋能/生态/赋权"]
- 禁止格式：超过 N 个 emoji，超过 X 字
- 禁止内容：具体数字承诺、竞品名称

### 第三层：自由区间（故意留白，不规定）
- 开头方式：自由
- 句子数量：够清晰就行
- 角度选择：选最有趣的

### 禁止
- 超过 15 条规则的 system prompt（触发相变）
- "必须出现什么"比"绝对不出现什么"更多</span></pre>
