        <h2>香农熵 · Shannon Entropy</h2>
        <div class="concept-subtitle">信息论 · 用于 Prompt 质量诊断</div>
      </div>
      <span class="domain-badge tag-info">信息</span>
    </div>

    <div class="block">
      <div class="block-label">核心公式</div>
      <div class="formula">H = −Σ p(x) · log₂p(x)

熵高 → 输出分布扁平 → prompt 有歧义
熵低 → 输出分布尖峰 → prompt 很明确

实操：同一个 prompt 跑 5-10 次，观察输出分散程度
不需要算公式，只需要看输出多样性</div>
    </div>

    <div class="block">
      <div class="block-label">四级熵诊断</div>
      <table class="decision-table">
        <tr><th>症状（跑 5 次后）</th><th>熵级别</th><th>根因</th><th>修复</th></tr>
        <tr>
          <td>结构都不一样，甚至有的用中文有的用英文</td>
          <td class="signal">极高熵</td>
          <td>prompt 完全歧义</td>
          <td class="action">重写，加格式约束</td>
        </tr>
        <tr>
          <td>结构相同，内容切角差异很大</td>
          <td class="signal">高熵</td>
          <td>任务范围模糊</td>
          <td class="action">加 few-shot examples</td>
        </tr>
        <tr>
          <td>结构内容相似，但细节不同</td>
          <td class="signal">中熵</td>
          <td>正常的创造性变化</td>
          <td class="action">如果这就是目标，可以接受</td>
        </tr>
        <tr>
          <td>几乎完全相同（95%+）</td>
          <td class="signal">低熵</td>
          <td>prompt 过于确定</td>
          <td class="action">如果需要多样性，放松约束</td>
        </tr>
      </table>
    </div>

    <div class="block">
      <div class="block-label">代码模式 — Prompt 熵诊断工具</div>
      <div class="code-header">
        <span>entropy-checker.ts</span>
        <span>Claude Code 脚本</span>
      </div>
<pre><span class="cm">// 用于在 Claude Code 中快速诊断 prompt 质量</span>
<span class="cm">// 运行：cc run check-entropy --prompt="your prompt"</span>

<span class="kw">import</span> { <span class="var">Claude</span> } <span class="kw">from</span> <span class="str">'@anthropic-ai/sdk'</span>;

<span class="kw">async function</span> <span class="fn">diagnosePromptEntropy</span>(
  prompt: <span class="kw">string</span>,
  n = <span class="num">5</span>
): <span class="kw">Promise</span>&lt;<span class="var">EntropDiagnosis</span>&gt; {
  <span class="kw">const</span> client = <span class="kw">new</span> <span class="var">Claude</span>();
  
  <span class="cm">// 并行跑 n 次</span>
  <span class="kw">const</span> outputs = <span class="kw">await</span> Promise.<span class="fn">all</span>(
    <span class="var">Array</span>.<span class="fn">from</span>({ length: n }, () =>
      client.messages.<span class="fn">create</span>({
        model: <span class="str">'claude-opus-4-5'</span>,
        max_tokens: <span class="num">500</span>,
        messages: [{ role: <span class="str">'user'</span>, content: prompt }],
        temperature: <span class="num">1.0</span>,  <span class="cm">// 使用默认温度，不要降温</span>
      })
    )
  );
  
  <span class="kw">const</span> texts = outputs.<span class="fn">map</span>(o => o.content[<span class="num">0</span>].text);

  <span class="cm">// 简单结构相似度：比较段落数、总字数、关键词</span>
  <span class="kw">const</span> structures = texts.<span class="fn">map</span>(t => ({
    paragraphs: t.<span class="fn">split</span>(<span class="str">'\n\n'</span>).length,
    wordCount: t.<span class="fn">split</span>(<span class="str">' '</span>).length,
    hasCode: t.<span class="fn">includes</span>(<span class="str">'```'</span>),
    hasNumberedList: <span class="str">/^\d+\./m</span>.<span class="fn">test</span>(t),
  }));

  <span class="cm">// 计算结构一致性（越接近 1 = 低熵）</span>
  <span class="kw">const</span> structuralConsistency = <span class="fn">calcConsistency</span>(structures);
  
  <span class="kw">return</span> {
    sampleOutputs: texts,
    structuralConsistency,         <span class="cm">// 0-1，越高越一致</span>
    entropyLevel: structuralConsistency > <span class="num">0.8</span> ? <span class="str">'low'</span>
                : structuralConsistency > <span class="num">0.5</span> ? <span class="str">'medium'</span>
                : <span class="str">'high'</span>,
    diagnosis: <span class="fn">diagnose</span>(structures, texts),
  };
}

<span class="cm">// 在 Claude Code 工作流中:</span>
<span class="cm">// cc run diagnose-entropy → 识别问题 → cc run fix-prompt → 再诊断</span></pre>
    </div>

    <div class="callout callout-tip">
      <strong>✓ 最快用法</strong>
      不需要代码。对任何 prompt，脑内运行 3 次，想象输出的结构差异有多大。差异大 → 高熵 → 需要加约束。这个直觉训练 2-3 次就会内化。
    </div>

    <div class="quick-ref">
      <h4>降熵工具箱（按效果排序）</h4>
      <ul>
        <li>指定输出格式（JSON/Markdown结构/固定段落数）</li>
        <li>加 2-3 个 few-shot examples（最有效）</li>
        <li>指定输出长度范围（字数或 token 数）</li>
        <li>指定视角/角色（"你是...的专家"）</li>
        <li>加负面示例（"不要像这样..."）</li>
      </ul>
    </div>

<pre><span class="cm">## Prompt 质量诊断（香农熵方法）

### 诊断信号（不需要代码，脑内判断）
- 同一 prompt 跑 3 次，结构都不一样 → 极高熵，需要重写
- 结构相同但内容切角差异大 → 高熵，加 few-shot examples
- 细节略有不同，结构一致 → 正常，可接受
- 几乎完全相同 → 低熵，如需多样性则放松约束

### 降熵工具（按效果排序）
1. 指定输出格式（JSON / 固定段落结构）
2. 加 2-3 个 few-shot examples（最有效）
3. 指定输出长度范围（字数上限）
4. 指定视角/角色

### 触发降熵操作的条件
- 同一 prompt 连续 3 次输出质量差异超过 30%
- 输出语言不稳定（时中时英）
- 每次结构都需要人工调整</span></pre>
