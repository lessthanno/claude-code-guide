Plan Mode 是 Claude Code 里最容易被误用的功能之一。

**它是干什么的：**

进入 Plan Mode 后，Claude 只规划、不执行。它会读取文件、搜索代码，然后给你一份实施方案——但不会改任何东西。

```bash
# 进入 Plan Mode
/plan

# 或者在对话中说
"先规划，不要动代码"
```

**什么时候用：**

1. **任务复杂、不确定改哪里**：先让 Claude 找到所有相关文件，确认影响范围，再动手。
2. **你想 review 方案**：跨多个文件的重构，先看路径对不对，再让它执行。
3. **风险高的操作**：数据库迁移、API 接口变更——Plan 完你来判断，不盲目执行。

**什么时候不用：**

- 明确的单文件改动：直接让它改，Plan 只是额外一轮对话
- 已经明确知道要改什么：Plan 会浪费你的时间

**怎么退出：**

```bash
# 退出 Plan Mode，回到正常执行模式
/exit-plan-mode

# 或者直接说
"好，按这个方案执行"
```

Plan Mode 的核心价值：**在你批准之前，零副作用地探索问题空间**。
