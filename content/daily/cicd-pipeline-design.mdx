CI/CD 不是"跑一下测试然后部署"，而是**在每次代码变更时，以可重复、可验证的方式，安全地将代码送到生产环境**。Claude Code 帮你设计真正工程化的流水线。

**从需求设计 GitHub Actions 工作流：**

```bash
claude "帮我设计这个项目的 GitHub Actions CI/CD 流水线：

项目：Go 后端 API + Next.js 前端
部署目标：AWS EC2（后端）+ Vercel（前端）
分支策略：
- main → 生产环境
- dev → 预发布环境
- feature/* → 只跑测试

需求：
1. PR 时：运行测试、lint、类型检查
2. 合并到 dev：部署到预发布
3. 合并到 main：部署到生产（需要手动审批）
4. 失败时发送 Slack 通知

给出完整的 workflow YAML 文件。"
```

**Go 后端的标准 CI 检查：**

```yaml
# .github/workflows/go-ci.yml
name: Go CI

on:
  pull_request:
    branches: [main, dev]
  push:
    branches: [main, dev]

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: testdb
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
        options: >-
          --health-cmd pg_isready
          --health-interval 10s

    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          
      - name: Run tests
        run: go test -race -coverprofile=coverage.out ./...
        env:
          DATABASE_URL: postgres://test:test@localhost/testdb
          
      - name: Check coverage threshold
        run: |
          COVERAGE=$(go tool cover -func coverage.out | grep total | awk '{print $3}' | tr -d '%')
          if (( $(echo "$COVERAGE < 70" | bc -l) )); then
            echo "Coverage $COVERAGE% is below 70% threshold"
            exit 1
          fi
```

CI/CD 的本质是工程信心：**你每次按下部署按钮时，知道什么会发生，为什么会发生，出了问题能快速诊断**。混乱的流水线让部署变成赌博；工程化的流水线让部署变成常规操作。
